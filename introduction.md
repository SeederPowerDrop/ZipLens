# ZipLens v1.3.0 — 사용자 안내서

> **ZipLens**는 macOS 네이티브 데스크톱 압축/해제 애플리케이션입니다.  
> 한글(CP949/EUC-KR) 파일명을 완벽하게 지원하며, 손상된 아카이브도 안전하게 처리합니다.

## 1. 프로그램 개요

ZipLens는 Tauri 2 프레임워크 기반의 크로스플랫폼 아카이브 관리 도구입니다.

### 핵심 특징

| 특징 | 설명 |
|------|------|
| **한글 파일명 완벽 지원** | CP949/EUC-KR로 인코딩된 ZIP 파일의 한글 파일명을 자동 감지하여 정상 표시 |
| **다중 포맷 지원** | ZIP, 7z, RAR, TAR, TAR.GZ, TAR.ZST, ISO, CAB, LZH, ALZ, EGG 등 |
| **선택적 추출** | 아카이브 내부를 미리보기하고 원하는 파일만 선택하여 추출 |
| **손상 파일 내성** | 일부 파일이 손상되어도 정상 파일은 계속 추출, 상세 보고서 제공 |
| **암호화 지원** | AES-256 및 ZipCrypto 암호화/복호화 지원 |
| **macOS 네이티브** | 투명 배경, vibrancy 효과, .app 번들 추출 지원 |
| **진행률 표시** | 실시간 프로그레스 바, 경과 시간, ETA 표시 |

## 2. 지원 포맷

### 압축 해제 (입력)

| 포맷 | 확장자 | 엔진 |
|------|--------|------|
| ZIP | `.zip` | Native (zip crate) + 7zz 폴백 |
| 7-Zip | `.7z` | Native (sevenz-rust) + 7zz 폴백 |
| RAR | `.rar` | 7zz sidecar |
| TAR | `.tar` | Native (tar crate) |
| TAR+GZIP | `.tar.gz`, `.tgz` | Native (flate2 + tar) |
| TAR+ZSTD | `.tar.zst`, `.tzst` | Native (zstd + tar) |
| ISO / CAB / LZH / ALZ / EGG | 각 확장자 | 7zz sidecar |

### 압축 (출력)

| 포맷 | 확장자 | 엔진 | 암호화 | 분할 압축 |
|------|--------|------|--------|-----------|
| ZIP | `.zip` | Native / 7zz (암호 설정 시) | ✅ AES-256, ZipCrypto | ✅ |
| 7-Zip | `.7z` | 7zz sidecar | ✅ AES-256 + 헤더 암호화 | ✅ |
| TAR+GZIP | `.tar.gz` | Native | ❌ | ❌ |
| TAR+ZSTD | `.tar.zst` | Native (멀티스레드) | ❌ | ❌ |

## 3. 압축 해제 (Extract)

### 방법 1: 드래그 앤 드롭

1. 압축 파일을 ZipLens 창 위에 **끌어다 놓습니다**.
2. 파일 목록이 미리보기로 표시됩니다.
3. 추출할 파일을 **체크박스로 선택**합니다.
4. 상단의 **"Extract Loaded Archive"** 버튼을 클릭합니다.
5. 추출 대상 폴더를 선택합니다.
6. 추출이 완료되면 **보고서**가 표시됩니다.

### 방법 2: 파일 탐색

1. 메인 화면의 **드롭 영역**을 클릭합니다.
2. 파일 선택 다이얼로그에서 압축 파일을 선택합니다.
3. 이후 과정은 드래그 앤 드롭과 동일합니다.

### 방법 3: CLI (우클릭 메뉴 / 터미널)

```bash
# 압축 해제 모드로 직접 실행
ZipLens --extract /path/to/archive.zip

# 또는 확장자 자동 인식
ZipLens /path/to/archive.zip
```

### 파일 충돌 처리

이미 동일한 이름의 파일이 존재할 경우, 세 가지 옵션이 제공됩니다:

| 옵션 | 동작 |
|------|------|
| **Overwrite (덮어쓰기)** | 기존 파일을 새 파일로 대체합니다 |
| **Append Suffix (접미사 붙이기)** | `파일명 (1).txt` 형태로 이름을 변경하여 저장합니다 |
| **Cancel (취소)** | 추출을 중단합니다 |

## 4. 압축 하기 (Compress)

### 기본 압축

1. 우측 패널에서 **출력 포맷** (`.zip`, `.tar.gz`, `.tar.zst`, `.7z`)을 선택합니다.
2. **Compress File** (파일 선택) 또는 **Compress Folder** (폴더 선택) 버튼을 클릭합니다.
3. 저장 위치를 선택합니다.

### 드래그 앤 드롭 압축

- 압축 파일이 **아닌** 일반 파일/폴더를 창에 드래그하면 자동으로 **압축 모드**로 전환됩니다.

### 암호 설정

1. **"Set Password (Optional)"** 체크박스를 활성화합니다.
2. 비밀번호를 입력합니다.
3. 암호화 방식을 선택합니다:
   - **AES-256 (Recommended)**: 강력한 암호화, 대부분의 해제 프로그램에서 지원
   - **ZipCrypto**: 레거시 호환용, 보안 수준이 낮음

### 분할 압축

**Split Archive** 입력란에 분할 크기를 지정합니다:

| 값 | 의미 |
|----|------|
| `0` 또는 비움 | 분할하지 않음 |
| `10m` | 10MB 단위로 분할 |
| `100m` | 100MB 단위 |
| `700m` | 700MB (CD 크기) |
| `4g` | 4GB (FAT32 제한) |

## 5. 파일 미리보기 및 선택

아카이브를 로드하면 파일 목록이 표시됩니다.

### 폴더 탐색

- **폴더명 클릭** → 해당 폴더 내부로 이동
- **📁 .. (Up to Parent)** → 상위 폴더로 이동
- **Breadcrumb 경로** → 원하는 깊이로 직접 이동

### 파일 선택 도구

| 버튼 | 기능 |
|------|------|
| **Select All** | 모든 파일을 선택합니다 |
| **Deselect All** | 모든 선택을 해제합니다 |
| **Invert Selection** | 선택/미선택 상태를 반전합니다 |

### 표시 정보

각 파일 항목에는 다음 정보가 표시됩니다:

- **체크박스**: 추출 여부 선택
- **확장자 배지**: 파일 타입 (예: `TXT`, `JPG`)
- **파일명**: 한글 포함 완전 디코딩된 이름
- **용량**: 추출 시 예상 크기
- **🔒 아이콘**: 암호화된 파일
- **⚠️ 아이콘**: 손상된 파일 (에러 메시지 포함)

### 상단 통계

```
125 / 5000 files selected
(Extracted Size: 2.3 MB / Total: 180.5 MB)
```

## 6. 암호화된 아카이브

### 암호가 필요한 파일 열기

1. 암호화된 아카이브를 로드하면 **패스워드 입력 모달**이 자동으로 표시됩니다.
2. 비밀번호를 입력하고 **Unlock** 버튼을 클릭합니다.
3. 잘못된 비밀번호일 경우 에러 메시지가 표시되며 재시도할 수 있습니다.
4. **Cancel** 버튼으로 취소할 수 있습니다.

### 비밀번호 보기 토글

- 비밀번호 입력란 우측의 **👁 아이콘**을 클릭하면 입력 내용을 확인할 수 있습니다.

## 7. 오류 처리 및 보고서

### 추출 중 파일 오류 발생 시

손상된 파일을 만나면 **오류 프롬프트**가 표시됩니다:

| 버튼 | 동작 |
|------|------|
| **무시** | 해당 파일을 건너뛰고 다음 파일로 진행 |
| **모두 무시** | 이후 모든 오류를 자동으로 건너뜀 |
| **중단** | 추출을 즉시 중단 |

### 추출 보고서

추출 완료 후 결과 보고서가 모달로 표시됩니다:

- **성공한 파일 수** 및 목록
- **실패한 파일 수** 및 각 파일의 에러 원인
- 파일명, 확장자, 전체 경로가 표시됩니다

### 보고서 내보내기

| 형식 | 버튼 | 내용 |
|------|------|------|
| **TXT** | Export TXT | 구조화된 텍스트 보고서 (파일명, 확장자, 용량, 경로, 에러) |
| **CSV** | Export CSV | 엑셀 호환 표 (상태, 파일명, 확장자, 용량, 전체경로, 에러) |

CSV 파일은 **UTF-8 BOM** 포함으로 저장되어, Excel에서 한글이 깨지지 않고 바로 열립니다.

#### CSV 칼럼 구조

```
상태,파일명,확장자,용량,전체경로,에러
성공,"가까워지는 것에 대한 두려움.txt","TXT","3.2 KB","소설(ㄱ)/가까워지는 것에 대한 두려움/...",
실패,"손상된파일.txt","TXT","0 B","소설(ㄴ)/손상된파일/...","ERROR: Data Error"
```

## 8. CLI 사용법

ZipLens는 커맨드라인 인수를 통해 직접 실행할 수 있습니다.

```bash
# 압축 해제 모드
ZipLens --extract /path/to/archive.zip

# 압축 모드
ZipLens --compress /path/to/file1 /path/to/file2

# 자동 감지 (확장자 기반)
ZipLens /path/to/archive.rar    # → 압축 해제 모드
ZipLens /path/to/folder         # → 압축 모드
```

### 자동 감지 지원 확장자

`zip`, `7z`, `rar`, `tar`, `gz`, `tgz`, `zst`, `tzst`, `cab`, `iso`, `lzh`, `alz`, `egg`

위 확장자를 가진 파일을 인수로 전달하면 자동으로 **압축 해제 모드**로 실행됩니다.  
그 외의 파일/폴더는 **압축 모드**로 실행됩니다.

## 9. 기술 사양

### 아키텍처

```
┌─────────────────────────────────┐
│       Frontend (WebView)        │
│  TypeScript + Vite + CSS        │
│  main.ts / ui.ts / utils.ts     │
├─────────────────────────────────┤
│       Tauri IPC Bridge          │
│  invoke() / listen()            │
├─────────────────────────────────┤
│       Backend (Rust)            │
│  archive.rs (core logic)        │
│  lib.rs (app setup)             │
├──────────┬──────────────────────┤
│ Native   │   7zz Sidecar       │
│ zip      │   (RAR, ISO, CAB,   │
│ tar      │    ALZ, EGG, LZH,   │
│ flate2   │    encrypted ZIP/7z) │
│ zstd     │                      │
│ sevenz   │                      │
└──────────┴──────────────────────┘
```

### 한글 인코딩 처리 흐름

```
ZIP 파일 내부 파일명 (raw bytes)
        │
        ▼
  ┌─ ASCII인가? ──────────→ 그대로 사용
  │
  ├─ EUC-KR 디코딩 시도
  │   └─ 한글 문자 포함? ──→ EUC-KR 결과 사용
  │
  └─ UTF-8 유효? ──────────→ UTF-8로 사용
        │
        ▼
     Lossy UTF-8 (최후 수단)
```

7zz sidecar 사용 시, macOS에서 발생하는 **PUA(Private Use Area) 문자 변환** 문제도 자동 복구합니다:

```
7zz 출력: PUA 문자 (0xEF00~0xEFFF)
        │
        ▼
  하위 바이트 추출 → CP949 raw bytes 복원
        │
        ▼
  EUC-KR 디코딩 → 정상 한글 파일명
```

### 의존성

| 분류 | 라이브러리 | 버전 | 역할 |
|------|-----------|------|------|
| 프레임워크 | Tauri | 2.x | 앱 프레임워크 |
| ZIP | zip crate | 2.2.0 | ZIP 읽기/쓰기 |
| GZIP | flate2 | 1.0 | gzip 압축/해제 |
| TAR | tar | 0.4.40 | tar 아카이브 |
| ZSTD | zstd | 0.13.0 | Zstandard 압축 |
| 7Z | sevenz-rust | 0.6.0 | 7z 네이티브 처리 |
| 병렬처리 | rayon | 1.8.1 | 멀티스레드 ZIP 추출 |
| 인코딩 | encoding_rs | 0.8 | CP949/EUC-KR 변환 |
| 사이드카 | 7zz | 24.09 | RAR 등 추가 포맷 |

## 10. 알려진 제한 사항

| 항목 | 설명 |
|------|------|
| **손상 감지 시점** | 아카이브의 목차(Central Directory)가 정상이면 열기 단계에서는 손상을 감지하지 못합니다. 실제 데이터 손상은 추출 시점에 발견됩니다. |
| **macOS .app 번들** | 암호화된 ZIP이거나 선택적 추출 시에는 ditto 대신 일반 추출을 사용하므로, 코드 서명이 깨질 수 있습니다. |
| **분할 압축** | ZIP과 7z 포맷에서만 지원됩니다. TAR 계열은 분할을 지원하지 않습니다. |
| **검색 기능** | 현재 검색 입력란은 UI에 존재하지만 필터링 기능은 아직 구현되지 않았습니다. |
| **정렬 기능** | 칼럼 헤더의 정렬 버튼은 UI에 존재하지만 정렬 로직은 아직 연결되지 않았습니다. |
